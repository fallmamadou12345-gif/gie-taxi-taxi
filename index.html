Hébergement gratuit avec disque persistant pour SQLite
1
Créer un compte Render
Aller sur render.com → Cliquer "Get Started for Free" → Se connecter avec votre compte GitHub (même email).
2
Créer un nouveau Web Service
Dashboard Render → "New +" → "Web Service" → Connecter votre repository GitHub gie-taxi-taxi.
3
Configurer le service
Remplir les champs comme ceci :
ChampValeur à mettre
Namegie-taxi-taxi
RegionFrankfurt (EU) — plus proche d'Afrique
Branchmain
RuntimeNode
Build Commandnpm install
Start Commandnode server.js
PlanFree (0$/mois)
4
Ajouter un Disque persistant (IMPORTANT pour SQLite)
Sans disque, les données se perdent à chaque redémarrage !
Dans le panneau "Disks" du service → "Add Disk" :
ChampValeur
Namegie-data
Mount Path/data
Size1 GB
Coût~7 USD/mois ≈ 4 200 FCFA/mois
⚠️
Le disque persistant est indispensable pour que les données (membres, cotisations, crédits) ne se perdent pas. C'est le seul coût de l'application : ~4 200 FCFA/mois.
5
Configurer les variables d'environnement
Onglet "Environment" → "Add Environment Variable" → Ajouter ces 3 variables :

JWT_SECRET = GieTaxiTaxi2026SenegalSecretKey!X7k
NODE_ENV = production
🔐
Changer JWT_SECRET par une longue chaîne aléatoire de votre choix. C'est la clé de sécurité de l'application.

7
Problèmes fréquents et solutions
Ce qui peut bloquer et comment résoudre
🔴
Application lente au premier chargement
Cause: Le plan Free de Render "dort" après 15 min d'inactivité. Premier chargement = 30-60 secondes.
Solution: Passer au plan Starter (7$/mois) ou utiliser UptimeRobot.com (gratuit) pour "pinger" l'app toutes les 10 minutes.
🔴
Données perdues après redéploiement
Cause: Oubli du disque persistant.
Solution: Vérifier que le Disk est bien configuré sur /data et que DB_PATH=/data/gie.db est défini.
🔴
Erreur "Cannot find module 'better-sqlite3'"
Cause: Build Node.js incompatible avec better-sqlite3.
Solution: Dans Render Settings → Node Version → mettre 18.19.0 exactement.
🔴
Membre ne peut pas se connecter
Cause: Le numéro de téléphone n'est pas dans la base de données.
Solution: Le Directeur ajoute le membre dans Gestion Membres → Nouveau Membre. PIN par défaut: 1234
🔴
L'application ne charge pas sur iPhone
Cause: Safari nécessite HTTPS pour les PWA.
Solution: S'assurer que Render a bien généré le certificat SSL (cadenas 🔒 dans le navigateur).
